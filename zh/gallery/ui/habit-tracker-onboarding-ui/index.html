习惯养成引导页提示词：Habit Tracker Onboarding

这条 习惯养成引导页提示词：Habit Tracker Onboarding 适合做移动端仪表盘、产品概念图、SaaS 头图和界面探索。 这个案例的主题重点包括 习惯、引导页。

Create a polished mobile onboarding screen for a habit tracker app with calm colors, progress indicator, habit cards, friendly illustration area, one primary button, and realistic spacing. No app store badges, no logos, no tiny text.

你可以怎么用

  • 直接复制提示词，用于生成界面设计方向的新图。
  • 替换主体或场景，保留布局清晰度、真实留白、层级和界面完成度这组质量要求。
  • 把标签 习惯、引导页、健康 当作后续扩展关键词。

这条提示词为什么有效

这条提示词先锁定习惯、引导页，再补充布局清晰度、真实留白、层级和界面完成度，最后用清晰结构和质量限制减少跑偏。用户复制后只需要替换主体、风格或比例，就能快速生成同类型图片。

适合的使用场景

这个案例更适合移动端仪表盘、产品概念图、SaaS 头图和界面探索。它最强的地方在于布局清晰度、真实留白、层级和界面完成度。

怎么改写成你自己的版本

换掉应用类型和核心工作流，但保留明确主操作和可信的导航结构。

模型与比例建议

适合使用能处理界面层级、留白和真实组件的模型；如果文字噪点太多，直接重生成。

移动端界面用 9:16，SaaS 头图用 16:9，仪表盘预览用 4:3。

可替换变量

  • 主体变量：把当前主题（习惯、引导页）替换成你的产品、人物、活动或场景。
  • 构图变量：保留布局清晰度、真实留白、层级和界面完成度，再调整近景、俯拍、居中构图、留白或多分镜。
  • 输出变量：按平台选择 1:1、4:5、9:16 或 16:9，并提前说明是否需要标题留白。

常见失败点

  • 忽略细小假文字，重点判断布局、留白和组件层级。
  • 如果导航或界面逻辑不成立，需要重生成。
  • 把结果当视觉 mockup，不要当最终 UI 文案。

推荐工作流

  • 先复制英文提示词并只替换主体，不要一次改太多变量。
  • 生成 3-5 张候选图，保留构图最稳定的一张。
  • 再根据目标平台改比例、裁切和标题留白。
