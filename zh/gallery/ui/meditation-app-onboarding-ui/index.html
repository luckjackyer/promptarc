怎么改写成你自己的版本

换掉应用类型和核心工作流，但保留明确主操作和可信的导航结构。

模型与比例建议

适合使用能处理界面层级、留白和真实组件的模型；如果文字噪点太多，直接重生成。

移动端界面用 9:16，SaaS 头图用 16:9，仪表盘预览用 4:3。

可替换变量

  • 主体变量：把当前主题（meditation、引导页）替换成你的产品、人物、活动或场景。
  • 构图变量：保留布局清晰度、真实留白、层级和界面完成度，再调整近景、俯拍、居中构图、留白或多分镜。
  • 输出变量：按平台选择 1:1、4:5、9:16 或 16:9，并提前说明是否需要标题留白。

常见失败点

  • 忽略细小假文字，重点判断布局、留白和组件层级。
  • 如果导航或界面逻辑不成立，需要重生成。
  • 把结果当视觉 mockup，不要当最终 UI 文案。

推荐工作流

  • 先复制英文提示词并只替换主体，不要一次改太多变量。
  • 生成 3-5 张候选图，保留构图最稳定的一张。
  • 再根据目标平台改比例、裁切和标题留白。
