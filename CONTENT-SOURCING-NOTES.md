# PromptArc Content Sourcing Notes

Last updated: 2026-05-22

## What Competitors Make Clear

- Strong prompt galleries lead with search, category browsing, image previews, and copy-ready prompt actions.
- Useful categories are outcome-based, not abstract: product photography, poster design, UI, infographic, photography, character, portrait, typography, and style tests.
- The content moat is volume plus organization. A small prompt tool without a connected model is weaker than a browsable library with clear examples.
- Premium or gated offers should come later, after the free library proves search demand and users save/copy specific categories.

## Current Expansion Rule

- Use only content we can present as PromptArc curated samples or properly attributed external sources.
- Rewrite prompts into original English descriptions instead of copying raw third-party prompt text.
- Avoid famous brands, celebrities, game characters, copyrighted character names, and recognizable logos unless there is a clear licensed or public-domain reason.
- Prefer categories with clear commercial or search intent: ecommerce product visuals, social posters, infographics, UI screens, photography references, and typography layouts.

## Next Collection Targets

- 100 safe English image prompts split across 8-10 clear categories.
- One lightweight detail page template per high-intent category, starting with product ads, infographics, photography, and poster design.
- A simple source-review field for each entry: original, curated, external open-source, or user-submitted.

## Generated Content Pipeline

- New prompts start in `content-pipeline/prompt-candidates.json`, not directly in the public gallery.
- Each prompt should generate 3-4 image candidates before review.
- Use `content-pipeline/review-rubric.json` to score outputs. Publish only when the total score is 82 or higher.
- Run `node scripts/audit-content-pipeline.mjs` before moving any generated image into `assets/gallery/`.
- Final public entries should use `sourceLabel: "PromptArc generated"` when the image is generated by our own pipeline.

## Original-First Cleanup Direction

- Long-term public gallery target: mostly or entirely `PromptArc generated` images.
- Existing external and legacy curated images should be treated as placeholders or references.
- Run `node scripts/audit-gallery-cleanup.mjs` to produce `content-pipeline/gallery-cleanup-report.json`.
- Regenerate useful old concepts into a consistent format instead of keeping mixed source images.
- Default public thumbnail ratio is `4:5`; UI examples may keep a `9:16` source image but should still display cleanly in the card crop.
