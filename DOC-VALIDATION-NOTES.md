# Documentation Validation Notes

This file translates the earlier handbooks into practical implementation guidance for this project.

## What remains valid

- GitHub Pages plus a custom domain is still a good low-cost launch path for a static MVP.
- Google Search Console should be connected as soon as the site is live.
- Long-tail SEO pages are more realistic for a new site than broad head terms.
- Monetization should start with intent-driven channels such as affiliate links and digital products.

## What was corrected

- Do not treat third-party SEO blog posts as proof that a workflow is officially supported or unchanged.
- Do not assume fixed free quotas for keyword tools. Pricing and query limits change often.
- Do not make AdSense the launch bottleneck for a brand-new site. It is slower and less controllable than an affiliate plus lead magnet setup.
- Do not publish outdated HARO instructions. Treat Featured or other current PR workflows as separate, current platforms.
- Do not hard-code affiliate rates or platform fees into evergreen documentation without pointing to the official pricing or policy page.

## What this project now does instead

- Launches a static tool site first
- Uses template pages for indexing
- Adds a configurable email capture flow
- Reserves clear spots for affiliate links and a Gumroad product
- Keeps external setup in configuration and checklists instead of burying it in the UI
